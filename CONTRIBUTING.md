# Contributing to the UIC Handbook

Thank you for your interest in improving the Umpire Coordinator Handbook! This project is designed as a template for youth baseball organizations to maintain high standards and operational continuity.

## How to Contribute

### Submitting Change Requests
If you have suggestions for improvement, please follow these steps:
1. **Open an Issue:** Describe the change you'd like to see. Is it a rule update, a new RefTown workflow, or a clarification?
2. **Propose the Text:** If possible, provide the updated text or a draft of the new section.
3. **Wait for Feedback:** We aim to maintain a "Veteran UIC" persona throughout the manual—authoritative yet supportive.

### Pull Requests & Branching
**This repository enforces a strict Feature Branch (FB) workflow.**
1. **No Direct Commits:** Committing directly to the `main` branch is strictly prohibited, even for repository owners.
2. **Branch Naming:** Create a new branch for every change (e.g., `fb/update-pitch-counts` or `fix/reftown-link`).
3. **The Review Loop:** All changes must be submitted via a Pull Request (PR) or Merge Request (MR).
4. **Sanitization Check:** Before merging, ensure no league-specific data (IDs, personal names, local locations) has been introduced. Use placeholders like `[League Name]` instead.

## Style Guidelines
- **Zero-Assumption:** Do not assume the reader knows the intricacies of RefTown or local board politics. Explain the "Why" behind the "How."
- **Resilient Linking:** Avoid deep-linking to specific articles that may break. Use portal-level links and provide navigation instructions.
- **Aesthetics:** If modifying HTML templates, ensure they remain optimized for browser-side printing (Ctrl+P).
